以下文章来源于德鼎创新 ，作者王岳华

德鼎创新 .

DraperDragon Innovation
本文作者德丰杰龙脉合伙人王岳华，在 2022 年 9 月 25 日由 TGO 鲲鹏会主办的 GTLC 全球技术领导力峰会·上海站，发表了题为《Web3 的趋势未来以及投资思考》的主题演讲，完整版演讲视频可浏览视频号。
早期投资必须思考未来，我们最常听到的一句话是未来已来，但到底未来是什么？未来会如何？确实难以判断预测。但是思考未来是一个必须经常提醒自己去做的事，想想下一年如何如何，5 年后又如何如何，10 年后呢？这个社会，经济，技术发展，商业应用，环境，甚至自己的生活可能有什么样的变化。思考未来是投资人与创业者所必须时常思考的事。近几年热门的话题无非是 Web3, DAO, 元宇宙等非常未来已来的话题。我们看看 2022 年 7 月份 Gartner 发布的对于区块链这个生态业态的一个细分领域的生命周期的一个预判，其中元宇宙从早期采用者到产业成熟期还要 10 年以上的时间，而 DAO 以及 Web3 则是还需要 5-10 年的时间。从早期投资的角度而言，现在就是投资人开始布局介入，创业者开始思考的时机。

互联网发展至今，中心化平台，中心化机构解决了很多问题，提高了效率，也促进了整个生态发展，特别是所谓的网络效应被中心化平台发挥的淋漓尽致。但是 Web2 解决了问题，也制造了新问题，用户身份的泄漏，数据的不透明，不自主，侵犯隐私甚至危及财产安全的数据窃盗。中心化平台的不对称优势，行业垄断等等，都使得用户不再自由，无法信任中心化平台。而创新的分布式数据库，智能合约，隐私计算，自主数据身份，分布式存储等等，就是要解决这些问题，让互联网更加的进步。所以我们迈向 Web3。除了 Web3 之外，DAO 以及元宇宙这三者将构建一个新未来的雏形，由这三者触发的更加创新的基础设施，解构传统的商业模式并再重构成新的经济组织体系以及新商业范式和模式都会使整个人类文明更加的进步。所以我们说 Web3 是未来的基础设施，DAO 是未来的经济组织形式而元宇宙是未来的生活形态。

现在市场上很多的项目都声称是 Web3 项目，但大都只是触及到了概念或涉及简单的技术基础设施，这些都是 Web3 的一个部分，但 Web3 却不仅仅如此。打个比喻就好比是盲人摸象。各说各话，都碰到了一点，但都无法一窥全貌，事实上，这个全貌也不成型，而是一直在演进创新变化。然而有三个基本要素驱动着 Web3 演进。第一个就是人工智能与智能合约技术，说到人工智能或许读者会有疑问这个不是老技术了吗，是的人工智能的发展至今已有三十多年但是一直在专用场景，比如视频识别，但是一定的程度的人工智能是 Web3 的基本基石，比如欧美常说的语意互联网 (Sematic Web), 空间互联网（Spatial Web），这些都是基于一定程度的人工智能才能成型。其实在所有的 Web3 应用场景里，所有的数据都需要被或强或弱的智能算法处理过，越是强的人工智能越能提供体验更好的 Web3 应用。第二是分布式计算，也可以说成是边缘计算，这个边缘计算的涵盖层面是更为广义的。因为 Web3 的数据来源是分布式的，隐私的计算也必须在地完成，每个分布式的节点担负的计算跟存储功能必须完善，一个 Web3 系统的数据才能被有效的处理，因此边缘计算是一个重要基础。第三是分布式的数据网络，这里提到的网络侧重在数据本身的结构，必须是基于类区块链协议，基于一定的共识机制，数据的来源，验证，核实都是基于分布式数据库的结构由代码自动实现。这三个是 Web3 的基本要素。试想 3 年后一辆自动驾驶的汽车，具备合适的人工智能所以可以完全的自动驾驶，这个车子本身就是一个充分的边缘计算节点，并可以完善的处理其他节点数出过来的数据，并将处理好的数据，或自身产生的原生数字也完善的发送出去，那么车主一个人看着改造过的可以像 AR 成像一样的挡风玻璃，沉浸在自己的元宇宙里。

Web3 的基本逻辑其实很单纯很直观，就是用户自主，也就等同于去中心化。所以很多人说 Web3 是去中心化，其实核心在于用户自主，然而如何从技术层面真正的实现用户自主，就需要很多创新的基础设施，一旦基础设施完备，原生的 Web3 应用才能蓬勃发展。所以 Web3 要解决的是如何做到完全的用户自主，并且在这个基础上，衍生新的商业模式，甚至新的商业业态。除了逻辑与基本要素，我们也有一个类比 Web3 的传统方法。那就是操作系统，Web3 的底层来自于分布式账本技术，分布式数据库技术，这个就好比是操作系统里的文档系统（Filing）以及 I/O（输出入系统），也是区块链里的 Layer 1， 处理数据的结构。分布式存储就是操作系统里面的文档系统（Filing）, 分布式计算机就更直接是操作系统里的 CPU 负责计算，分布式的数据传输（分布式通信）也就好比 I/O。CPU, Filing System (文档系统)， I/O（输出入）都是操作系统的基本要素，类比到 Web3 的底层亦是如此，然而在未来我们是否有机会看到一个大一统的协议来构建 Web3 呢？笔者认为是不必要的，Web3 本来就该是各个功能模块协议化，分层可组合，就像乐高一样，只要你有足够的组件，你就能够完成一个成品 / 产品。

除了以上所说的底层基础设施，Web3 有另外两个在过去难以触碰的关键，一个是去中心化数字身份，另一个是隐私计算。这两个中间件是息息相关的。数字身份并不是新的话题，Web3 领域里面，DID（去中心化身份）是一个必要的基本，就好比一个人没有身份证一样，哪儿也去不了，什么也不是，没有任何个人机构可以识别你，认可你。所以 ID 是一个最关键的基础，没有 ID，就没有任何资产。在原生 Web3 的网络中，ID 一般是由钱包地址来决定，但显然钱包地址不是唯一的 ID，去中心化的身份本来就允许多重的 ID, 可以拆解并重组，但是每个 ID 的 reputation（影响力）跟 credit（信任度）必须由这个 ID 所参与交互的数据来验证认可核实，最终归属到这个 ID。所以即便是多重 ID，ID 的排列组合与融合也可以创造出不同的应用场景与商业模式。除了 Web3 原生的 ID 之外，传统 Web2 的 ID 也必须进行升级迭代，加上一层隐私保护与隐私计算的保护。隐私计算早在 1985 年就由 S Goldwasser 提出了零知识证明技术，1987 年 Oden Goldreich 等人在基于清华大学姚期智教授的两方安全计算基础上发展了多方安全计算，在加上同态加密以及联邦学习等算法，进一步讲将隐私计算应用在数据保护的场景，除了 Web3 原生场景的隐私保护之外，在 Web 领域的医疗，金融，政务，供应链等等也开始了商业落地案例。

那么 Web3 的基本技术架构清晰了以后，未来的趋势就是建构在这些底层技术上的应用，而且所有应用上的数据都是分布式数据网络所提供处理的数据，进而形成今天我们所知道的 NFT，DeFi, 元宇宙，创作者经济，去中心化媒体，去中心化社交，去中心化信用评比等等 DApps (去中心化应用)。在 Web3 的生态体系里面，还有一个非常关键的就是新的组织形态，现在看起来，DAO 就是一个很 Web3 的架构来发展新的组织形态，DAO 的核心逻辑是把所谓的 Share Holder 概念转变成 Stake Holder, 也可以说是人人为我，我为人人， 而其功能以及属性可以透过智能合约的规定以及部署来随着这个组织要完成的目标来变化，其中的智能合约就好像堆乐高积木一样，由 DAO 的成员根据规则一个一个加起来。当然每个成员的权责利根据贡献，信用评比，参与，等等不同维度来建立自己的灵魂绑定。

最后， Web3 塑造未来，核心逻辑是去中心化，在实际场景体现出来就是用户自主，但是把思路再开点脑洞，或许 Web3 是在追求我们所不知道的不知道的领域。传统上你知道你所知道的就是所谓的知识，你知道你不知道的，那可以用 Web2 的百度，谷歌去查询。你不知道你知道的，也没关系，Web2 里的亚马逊，脸书可以依靠人工智能帮你知道。再来就是你不知道你不知道的，可能 Web3 的机会就在这里。期待与大家一起共创未来。
